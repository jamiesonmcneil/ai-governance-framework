# Acme Organization AI Governance Rules (Org Layer)

These rules are **additive** to the immutable Core rules. They cannot override or weaken any Core rule.

1. All AI sessions must use an approved enterprise instance (no public consumer models for sensitive work).
2. Any code generated by AI must undergo human review and automated testing before merging.
3. Prompt engineering must follow Acme Prompting Standards v1.2.
4. All production changes require explicit "CONFIRM PRODUCTION" + team lead approval.
5. Credential handling must follow the stricter of Core or Org policies.
6. PII and sensitive data must be redacted or masked before any AI context is sent.
